# 进度快照 4 - 后端多服务器连接完成

## 已完成内容
- 修改 `libs/hbb_common/src/config.rs`：
  - 支持从 `server-profiles` 读取启用的 ID Server 列表。
  - `get_rendezvous_server()` 会优先使用 profile 列表中的当前选中服务器，否则使用第一台启用服务器。
  - `get_rendezvous_servers()` 会返回 profile 中所有启用服务器。
  - `update_latency()` 改为基于 `auto-server-switch-threshold-ms` 判断，默认 `300ms`，避免低延迟时频繁切换。
- 修改 `src/common.rs`：
  - 新增 `get_key_for_server(server, sync)`，优先读取对应 profile key，找不到再回退全局 key。
  - TCP proxy 握手改为按当前服务器取 key。
- 修改 `src/client.rs`：
  - 新建连接拿到最终 rendezvous server 后，改用该 server 对应 profile key。
  - health check 连接改用当前 rendezvous server 对应 profile key。
- 修改 `src/rendezvous_mediator.rs`：
  - TCP rendezvous 注册握手改用当前 profile key。
  - relay server 选择优先使用当前 profile 的 relay server。
- 修改 `src/server.rs`：
  - 受控端创建 relay 连接时，按 relay/profile 查找 licence key。

- 当前阶段：模块 3 - 后端多服务器连接。
- 下一阶段将进入模块 4：Flutter 设置页卡片 UI。

## 具体修改的代码前后对比
### 1. 控制端连接 key 选择

```diff
修改前：
async fn _start_inner(..., key: String, ...)

修改后：
async fn _start_inner(..., mut key: String, ...)
let profile_key = crate::server_profiles::key_for_server(&rendezvous_server);
if !profile_key.is_empty() {
    key = profile_key;
}
```

### 2. 公共 key helper

```diff
修改前：
pub async fn get_key(sync: bool) -> String { ... }

修改后：
pub async fn get_key(sync: bool) -> String { ... }

pub async fn get_key_for_server(server: &str, sync: bool) -> String {
    let key = crate::server_profiles::key_for_server(server);
    if key.is_empty() {
        get_key(sync).await
    } else {
        key
    }
}
```

### 3. relay key 选择

```diff
修改前：
let licence_key = crate::get_key(true).await;

修改后：
let licence_key = crate::get_key_for_server(&relay_server, true).await;
```

### 4. hbb_common 多服务器选择

```diff
修改前：
custom-rendezvous-server 非空时，get_rendezvous_servers() 只返回单个自定义服务器。

修改后：
如果 server-profiles 非空，则优先返回 profiles 中所有启用服务器；
只有 profiles 为空时才回退 custom-rendezvous-server / 内置服务器。
```
