# 18-安装后升级提示修复完成

## 已完成内容
- 读取最新接力基线：`进度快照/17-中继首选服务器功能完成.md`。
- 创建本阶段源码备份：
  - `backup/11-安装后升级提示修复阶段/`
- 定位安装后提示原因：
  - 首页提示来自 `mainIsInstalledLowerVersion()`。
  - 点击“点击这里升级”会调用 `mainUpdateMe()`，实际进入内置 `--install` 流程。
  - 当前 Inno 安装器写的是自己的卸载项，但运行时版本检测读取 `rustdesk2.0` 内置安装项，缺少 `BuildDate` 时会误判为低版本。
  - 内置安装流程会删除安装目录后复制文件，和 Inno 安装方式混用时可能导致 Flutter `data` 目录缺失，从而无法运行。
- 修复运行时检测：
  - 自定义客户端不再显示“安装版本低，点击升级”卡片。
  - `BuildDate` 为空时不再判定为低版本。
  - 自定义客户端调用 `update_me` 时直接跳过内置升级流程。
- 修复 Windows 安装器：
  - Inno 安装包补写运行时读取的 `InstallLocation`。
  - Inno 安装包补写运行时读取的 `BuildDate`。
- 已重新构建 Windows Release 和 EXE 安装包。
- 已重新生成便携 ZIP，避免 Release 附件仍使用旧 DLL。

## 当前修改到哪个模块
- 当前模块：Windows 安装/升级提示与安装包注册表信息。
- 当前状态：代码、构建和安装包已完成。

### 2. `src/ui_interface.rs`

修改前：

```rust
pub fn update_me(_path: String) {
    goto_install();
}
```

修改后：

```rust
pub fn update_me(_path: String) {
    if crate::common::is_custom_client() {
        log::info!("Skip built-in update flow for custom client");
        return;
    }
    goto_install();
}
```

修改前：

```rust
let b = crate::platform::windows::get_reg("BuildDate");
return crate::BUILD_DATE.cmp(&b).is_gt();
```

修改后：

```rust
if crate::common::is_custom_client() {
    return false;
}
let b = crate::platform::windows::get_reg("BuildDate");
return !b.is_empty() && crate::BUILD_DATE.cmp(&b).is_gt();
```
