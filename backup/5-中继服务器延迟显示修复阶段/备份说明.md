# 5-中继服务器延迟显示修复阶段

## 阶段目标
- 修复服务器设置卡片中 `Relay Server` 没有延迟数值的问题。
- 点击刷新按钮后，ID Server 与 Relay Server 的延迟都应重新测试并回填到卡片。

## 修改前源码备份
- `src/server_profiles.rs`
- `flutter/lib/common.dart`
- `flutter/lib/mobile/widgets/dialog.dart`
- `flutter/test/server_settings_dialog_test.dart`

## 修改前问题定位
- 后端 `test_profiles_latency_json` 只测试 `profile.id_server`。
- 前端 `ServerProfileLatency` 只保存一个 `latencyMs` 字段。
- 卡片只在右上角显示整体状态，没有给 `Relay Server` 行显示独立延迟。

async fn test_endpoint_latency(server: &str, default_port: i32) -> (i64, String) {
    if server.is_empty() {
        return (-1, String::new());
    }
    let start = Instant::now();
    match connect_tcp(check_port(server, default_port), CONNECT_TIMEOUT).await {
        Ok(_) => (start.elapsed().as_millis() as i64, String::new()),
        Err(err) => (-1, err.to_string()),
    }
}
```

({String text, Color color})? _serverProfileRelayStatus(
  ServerProfileConfig profile,
  ServerProfileLatency? latency,
) {
  if (!profile.enabled || profile.relayServer.isEmpty) {
    return null;
  }
  if (latency == null) {
    return (text: translate('Testing'), color: Colors.grey);
  }
  if (latency.relayLatencyMs >= 0) {
    final color = latency.relayLatencyMs > 300 ? Colors.orange : Colors.green;
    return (text: '${latency.relayLatencyMs} ms', color: color);
  }
  return (text: translate('Offline'), color: Colors.red);
}
```
