# 备份说明 - 3-Rust构建bindgen修复阶段

## 创建时间
2026-06-30

## 阶段目标
修复 `libs/scrap` 在 Windows + LLVM 22 环境下 bindgen 生成 `vpx_codec_enc_cfg` / `aom_codec_enc_cfg` 为不透明结构体的问题，恢复 Rust 构建校验。

## 备份范围
- `Cargo.lock`
- `libs/scrap/Cargo.toml`
- `libs/scrap/build.rs`
- 本阶段后续还涉及 no-default 编译修复：
  - `src/server_profiles.rs`
  - `src/common.rs`
  - `src/server/audio_service.rs`
  - `src/server.rs`

## 修改前状态
- `libs/scrap/Cargo.toml` 使用 `bindgen = "0.65"`。
- `libs/scrap/build.rs` 直接使用 `bindgen::builder()` 生成 `vpx_ffi.rs`、`aom_ffi.rs`、`yuv_ffi.rs`。
- 当前生成结果中 `vpx_codec_enc_cfg` / `aom_codec_enc_cfg` 仅包含 `_address: u8`，导致 Rust 代码访问 `g_w`、`g_h` 等字段失败。

## 修改后对比
- `libs/scrap/Cargo.toml`

修改前：
```toml
bindgen = "0.65"
```

修改后：
```toml
bindgen = "0.72"
```

- `Cargo.lock`

修改前：`scrap` 依赖 `bindgen 0.65.1`。

修改后：`scrap` 依赖 `bindgen 0.72.1`，解决 LLVM 22/libclang 下 `vpx_codec_enc_cfg`、`aom_codec_enc_cfg` 被生成成 opaque struct 的问题。

- `src/server_profiles.rs`

修改前：
```rust
#[tokio::main(flavor = "current_thread")]
```

修改后：
```rust
use hbb_common::tokio;

#[hbb_common::tokio::main(flavor = "current_thread")]
```

- `src/common.rs`

修改后新增 no-default fallback：
```rust
#[cfg(not(any(feature = "use_dasp", feature = "use_samplerate")))]
pub fn audio_resample(
    data: &[f32],
    _sample_rate0: u32,
    _sample_rate: u32,
    _channels: u16,
) -> Vec<f32> {
    data.to_vec()
}
```

- `src/server/audio_service.rs`

修改前：
```rust
T: cpal::SizedSample + dasp::sample::ToSample<f32>,
let buffer: Vec<f32> = data.iter().map(|s| T::to_sample(*s)).collect();
```

修改后：
```rust
T: cpal::SizedSample,
f32: cpal::FromSample<T>,
let buffer: Vec<f32> = data.iter().map(|s| f32::from_sample(*s)).collect();
```

- `src/server.rs`

修改前：
```rust
crate::check_port(relay_server, RELAY_PORT)
let licence_key = crate::get_key_for_server(&relay_server, true).await;
```

修改后：
```rust
crate::check_port(&relay_server, RELAY_PORT)
let licence_key = crate::get_key_for_server(&relay_server, true).await;
```
